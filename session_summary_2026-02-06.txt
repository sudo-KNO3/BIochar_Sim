class OffSpecTrigger(Enum):
    """Triggers for off-spec load diversion."""
    HIGH_FOG = "high_fog"
    TRASH_HEAVY = "trash_heavy"
    UNKNOWN_CHEMISTRY = "unknown_chemistry"

class OffSpecParams:
    # Tank sizing
    tank_volume_m3: float = 12.0           # Product: 1 truck
    tank_volume_hub_m3: float = 24.0       # Hub: 2 trucks
    
    # Enabled triggers
    enabled_triggers: Tuple[str, ...] = (
        "high_fog", "trash_heavy", "unknown_chemistry", "visual_flag"
    )
    
    # Thresholds
    fog_threshold_mg_L: float = 500.0
    ts_high_threshold: float = 0.06
    ts_low_threshold: float = 0.01
    
    # Hold and disposition
    max_hold_days: float = 7.0
    haul_off_cost_per_m3: float = 85.0
    
    # Blend-back
    blend_back_rate_fraction: float = 0.10
    blend_back_requires_test: bool = True
    
    # Logging
    notify_on_diversion: bool = True
    log_all_decisions: bool = True
    
    def get_tank_volume(self, is_hub: bool = False) -> float:
        return self.tank_volume_hub_m3 if is_hub else self.tank_volume_m3
    
    def is_trigger_enabled(self, trigger: str) -> bool:
        return trigger in self.enabled_triggers

## POTENTIAL REVIEW CONCERNS

### 1. Type Consistency
- `OffSpecTrigger` is an Enum but `enabled_triggers` in `OffSpecParams` uses `Tuple[str, ...]` not `Tuple[OffSpecTrigger, ...]`
- **Rationale**: Allows JSON serialization without enum conversion; `is_trigger_enabled()` accepts string

### 2. Duplicate DeploymentMode/Subsystem Enums
- `facility_design.py`, `facility_geometry.py`, and `fmea.py` each define their own `DeploymentMode`/`Subsystem`
- **Handled**: Aliased imports in `__init__.py` (`FD_DeploymentMode`, `FG_DeploymentMode`)
- **Status**: Semantically identical but separate — consider future consolidation

### 3. Equipment Tag Consistency
- Product mode: TK-101, TK-102, etc.
- Hub mode: TK-201A, TK-201B, TK-202, etc.
- **Rationale**: Intentional separation; hub uses 200-series tags

### 4. Frozen Dataclass Mutability
- `DesignGap` is NOT frozen (mutable) while most other dataclasses are frozen
- **Rationale**: Allows updating status as gaps are resolved

### 5. CLI Entry Point Pattern
- Uses `sys.argv` directly instead of `argparse`
- **Rationale**: Matches existing module patterns in codebase

### 6. Conservative Envelope ≠ Equipment Placement
- `PRODUCT_REACTOR_ENVELOPE` (6.0×2.5×5.0m) ≠ PY-101 placement (4.5×2.5×5.0m)
- **Rationale**: Envelope is the *allowable maximum* for layout planning; actual equipment may be smaller
- Tests verify equipment fits *within* envelope, not exact match

### 7. `from __future__ import annotations` in facility_geometry.py
- Added to resolve forward reference issue (functions defined before `EquipmentPlacement` class)
- **Impact**: All type annotations in this module are now strings at runtime (PEP 563 behavior)
